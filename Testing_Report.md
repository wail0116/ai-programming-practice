## 4. 测试反思

### 已覆盖
- 核心播放流程正常（初始化→播放→暂停→继续→停止→切歌）
- 4 种播放模式逻辑正确
- 画面调整 4 个参数实时生效
- 截图和视频信息功能正常
- 配置持久化在应用重启后正确恢复

### 未覆盖（留给后续）
- **并发播放** — 计划中但尚未实现的多 slot 功能
- **不同编码格式兼容性** — 仅测试了 H.264 MP4/MKV，未覆盖 H.265/VP9/AV1
- **超长视频寻址** — 未测试超过 2 小时的视频在 90% 后的 seek 精度
- **异常文件** — 损坏视频文件的容错行为
- **UI 自动化** — 所有测试为手动操作，未使用 WinForms UI Automation 框架

### AI 辅助测试的体会
- Claude生成的测试清单覆盖了播放器的大部分功能点，但在实际操作中需要一一验证才能确认结果
- 边界条件测试（空路径、重复添加、文件被删除）的设计由 AI 提出，人类较容易遗漏
- 视觉类测试（画面调整效果）必须人工判断，AI 只能验证 API 调用是否抛异常
